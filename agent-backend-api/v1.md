## Invoke (SSOT semantic: always HTTP 200)

### `POST /v1/skills/{name}:invoke`
Request:
```json
{
  "input": { },
  "trace": { }
}
```

Sync success (HTTP 200):
```json
{ "ok": true, "output": { } }
```

Async accepted (HTTP 200):
```json
{ "ok": true, "job_id": "..." }
```

Failure (HTTP 200):
```json
{
  "ok": false,
  "error": { "code": "string", "message": "string", "retryable": false }
}
```

Required error codes (v0):
- `SKILL_NOT_FOUND` (unknown skill)
- `INVALID_JSON` (request body not valid JSON)
- `INVALID_INPUT` (skill-specific validation failure)
- `INTERNAL` (unexpected server error)

## Error Codes Reference

### Common Error Codes
- `SKILL_NOT_FOUND` - Skill does not exist
- `INVALID_JSON` - Request body is not valid JSON
- `INVALID_INPUT` - Missing or invalid parameters
- `INTERNAL` - Server internal error

### PR Error Codes
- `REPO_NOT_FOUND` - Course repository not found
- `TOML_SCHEMA_ERROR` - Invalid TOML structure
- `RENDER_FAILED` - README rendering failed

### COS Error Codes
- `STORAGE_ERROR` - COS operation failed
- `FILE_TOO_LARGE` - File exceeds size limit
- `QUOTA_EXCEEDED` - Daily quota exceeded

### MCP Error Codes
- `MCP_SERVER_NOT_FOUND` - MCP server not registered
- `MCP_TOOL_NOT_FOUND` - Tool not found in server
- `MCP_EXECUTION_ERROR` - Tool execution failed
