## 统一搜索入口: search

```
search
    │
    ├─ rag → 向量数据库
    ├─ brave → 网页搜索
    ├─ annas → 文献搜索
    ├─ arxiv → 论文搜索
    ├─ github → GitHub搜索
    ├─ cos → COS文件
    ├─ course → 课程搜索
    ├─ course_read → 课程README

```

## 单个列出的 Skills

### PR 相关

| Skill | 功能 |
|-------|------|
| `pr.preview` | 预览课程改动 |
| `pr.submit` | 提交PR |
| `pr.lookup` | 查看PR状态 |

运行要点：
- `pr.preview`、`pr.submit`、`pr.lookup` 调用 pr-server 时会透传 `PR_SERVER_TOKEN`（Bearer）。
- `pr.submit` 兼容 `idempotency_key`：客户端可传；未传时由 agent-backend 自动生成，满足 pr-server 的幂等要求。
- `pr.submit` 输出字段与 pr-server 新响应对齐：`pr_number <- data.pr.number`，`pr_url <- data.pr.url`，`branch <- data.pr.head_branch`。
- `pr.lookup` 兼容两种入参：`number`（推荐）与 `pr`（兼容旧调用）。

**pr.preview 请求**
```json
{
  "skill": "pr.preview",
  "input": {
    "campus": "shenzhen",
    "course_code": "COMP1011",
    "ops": [{"op": "add_lecturer_review", "lecturer_name": "张三", "content": "很好"}]
  }
}
```

## RAG 相关 Skills

### `rag.ingest`
RAG数据接入（异步）。

**请求**
```json
{
  "skill": "rag.ingest",
  "input": {
    "repo": "HIT-A/HITA_RagData",
    "ref": "main",
    "path_prefix": "optional/subdir",
    "collection": "hita_knowledge",
    "dry_run": false,
    "max_files": 100,
    "max_chunks": 5000
  }
}
```

### `rag.ingest_from_github`
从GitHub仓库批量下载并接入RAG（异步）。

**请求**
```json
{
  "skill": "rag.ingest_from_github",
  "input": {
    "repos": [
      {"org": "owner", "repo": "repo-name", "ref": "main"}
    ],
    "max_file_size": 10485760,
    "max_repo_size_mb": 500,
    "convert_to_md": true,
    "push_to_github": true,
    "target_repo": "HIT-A/HITA_RagData",
    "store_in_cos": true
  }
}
```

### `rag.sync_to_repo`
同步RAG数据到目标仓库（异步）。

**请求**
```json
{
  "skill": "rag.sync_to_repo",
  "input": {
    "repo": "HIT-A/HITA_RagData",
    "branch": "main",
    "daily": false
  }
}
```

兼容说明：
- `repo` 等价于 `target_repo`，`branch` 等价于 `target_branch`。
- 仍支持传入完整 `sources`、`target_repo`、`target_branch` 配置。

运行要点：
- Git 克隆/拉取/推送优先使用 `GITHUB_TOKEN`（可选 `GITHUB_USERNAME`，默认 `x-access-token`）。
- 任务关闭交互式凭据提示（`GIT_TERMINAL_PROMPT=0`），避免后台任务卡住。
- 提交阶段会自动设置仓库本地身份：`GIT_AUTHOR_NAME` / `GIT_AUTHOR_EMAIL`（未配置则使用默认值）。
